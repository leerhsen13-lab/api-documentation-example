# Спецификация REST API: Сервис проверки баланса банковской карты

Данный документ содержит техническое описание метода для интеграции мобильного приложения с внутренним банковским процессингом для получения актуального баланса и истории последних транзакций по карте клиента.

## Метод и URL-адрес
* **HTTP Метод:** GET
* **URL:** `/api/v1/cards/balance`
* **Формат данных:** JSON
* **Авторизация:** Bearer Token (в заголовках запроса)

---

## 1. Входные параметры (Query Parameters)

Параметры передаются в строке запроса (URL).


| Параметр     | Тип данных | Обязательный | Описание                                           | Пример             |
| :-------------| :-----------| :-------------| :---------------------------------------------------| :-------------------|
| `card_token` | String     | Да           | Уникальный зашифрованный токен карты клиента.      | `9876543210abcdef` |
| `currency`   | String     | Нет          | Валюта баланса (RUB, USD, EUR). По умолчанию: RUB. | `RUB`              |

---

## 2. Успешный ответ сервера (Response 200 OK)

В случае успешной обработки запроса сервер возвращает код `200 OK` и тело ответа в формате JSON.

### Пример ответа:
```json
{
  "status": "success",
  "card_id": "987654321",
  "balance": 15450.50,
  "currency": "RUB",
  "last_transactions": [
    {
      "date": "2026-05-30T14:20:00Z",
      "merchant": "Supermarket Magnit",
      "amount": -1250.00
    },
    {
      "date": "2026-05-30T10:15:00Z",
      "merchant": "Yandex Transport",
      "amount": -60.00
    }
  ]
}
```

### Описание полей ответа (JSON Schema):
* `status` (string) — Статус выполнения операции (`success`).
* `card_id` (string) — Маскированный идентификатор карты в системе.
* `balance` (number) — Доступный остаток средств на карте (разделитель — точка).
* `currency` (string) — Код валюты счета.
* `last_transactions` (array) — Массив (список) последних операций по карте.
  * `date` (string) — Дата и время транзакции в формате ISO 8601.
  * `merchant` (string) — Точка продаж (место списания).
  * `amount` (number) — Сумма операции (отрицательное значение означает списание).

---

## 3. Возможные ошибки (Error Responses)

* **400 Bad Request:** Передан неверный формат токена или неподдерживаемая валюта.
* **401 Unauthorized:** Не передан или устарел токен авторизации в Headers.
* **404 Not Found:** Карта с указанным `card_token` не найдена в системе.

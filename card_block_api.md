# Спецификация REST API: Сервис блокировки банковской карты.

Данный документ содержит техническое описание метода блокировки банковской карты клиента в случае утери, кражи или по желанию пользователя.

## Метод и URL-адрес
* **HTTP Метод:** POST
* **URL:** `/api/v1/cards/block`
* **Формат данных:** JSON 
* **Авторизация:** Bearer Token (в заголовках запроса)

---

## 1. Тело запроса (Request Body)

Параметры передаются в теле запроса в формате JSON.

### Пример тела запроса:
```json
{
  "card_token": "9876543210abcdef",
  "reason": "lost_card",
  "comment": "Потеряла карту в парке, прошу заблокировать"
}
```

### Описание параметров запроса:
| Параметр | Тип данных | Обязательный | Описание | Пример |
|:----------|:----------|:----------|:----------|:----------|
| `card_token` | String | Yes | Зашифрованный токен карты | `9876543210abcdef` |
| `reason` | String | Yes | Код причины блокировки (`lost_card` - утеря, `stolen` - кража, `client_request` - запрос клиента). | `lost_card` |
| `comment` | String | No | Текстовый комментарий от клиента | `Потеряла карту в парке` |

---

## 2. Успешный ответ сервера (Response 200 OK)

В случае успешной обработки запроса и блокировки карты сервер возвращает код `200 OK` и тело ответа в формате JSON.

### Пример ответа:
```JSON
{
  "status": "blocked",
  "card_id": "123456789",
  "updated_at": "2026-05-30T15:00:00Z"
}
```

### Описание полей ответ (JSON Schema):
* `status` (string) - Текущий статус карты в системе после операции (`blocked`).
* `card_id` (string) - Маскированный номер или ID заблокированной карты.
* `updated_at` (string) - Точное время изменения статуса карты в формате ISO 8601.

## 3. Возможные ошибки (Error Responses)

* **400 Bad Request:** Не передан обязательный параметр `card_token` или указана невалидная причина `reason`.
* **401 Unauthorized:** Неверный или просроченный токен авторизации в Headers.
* **404 Not Found:** Карта с указанным `card_token` не найдена в банковской системе.
